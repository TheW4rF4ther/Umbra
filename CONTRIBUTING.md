# Contributing to Blackbox Umbra

## ⛔ This Project Does NOT Accept Pull Requests or Contributions

**Blackbox Umbra is a proprietary commercial tool.** It is not an open-source project, and it does not accept external contributions, pull requests, feature requests, or bug reports from the public.

### Who Can Modify This Software?

Only **authorized employees and contractors of Blackbox Intelligence Group LLC** can modify, extend, or maintain this codebase. Any modifications must:

1. Be approved by authorized management
2. Follow internal code review processes
3. Maintain backward compatibility with existing engagements
4. Receive priority security auditing
5. Be documented in internal changelogs only

### What You CAN Do

✅ **Use** the software for authorized penetration testing engagements  
✅ Report security vulnerabilities to **security@blackboxintelgroup.com** (if authorized)  
✅ Request features through your Blackbox Intelligence Group LLC contact  
✅ Share feedback with your team lead or manager

### What You CANNOT Do

❌ Fork this repository  
❌ Submit pull requests  
❌ Create derivative works  
❌ Redistribute modified versions  
❌ Use code from this project in competing products  
❌ Contribute patches or enhancements via GitHub  
❌ Issue feature requests or pull requests  

### Legal Basis

This restriction is enforced by the **LICENSE** file (proprietary terms). Violation may result in:
- Legal action
- Termination of employment/contract
- Civil damages
- Criminal prosecution (in cases involving theft of trade secrets)

## For Internal Team Members

If you are an authorized Blackbox Intelligence Group LLC team member:

1. **Report bugs** to your technical lead or security@blackboxintelgroup.com
2. **Request features** through internal project management (Jira / GitHub Discussions for internal use only)
3. **Submit code** via internal code review process (not via GitHub pull requests)
4. **Access branches** via internal repository access controls

---

## Security Concerns

If you discover a **security vulnerability** in Blackbox Umbra:

1. **DO NOT** disclose it publicly or on GitHub
2. **DO** report it directly to **security@blackboxintelgroup.com**
3. Include: vulnerability description, reproduction steps, potential impact, suggested fix (if you have one)
4. Allow 48 business hours for initial response
